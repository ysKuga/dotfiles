# Karpathy Guidelines

## Think Before Coding
- State assumptions. Ask when unclear. Never run on wrong premise.
- Surface inconsistencies. Present tradeoffs. Push back when needed.
- No sycophancy. Ask only if ambiguous AND high-impact. Otherwise state assumption and proceed.

## Simplicity First
- Min code. YAGNI. No speculative abstractions.
- 100 lines beats 1000 lines. Always.
- No bloat. No dead code left behind.

## Surgical Changes
- Touch only what task requires. Nothing else.
- No side-effect edits to unrelated comments/code.
- Clean up dead code created by your changes.

## Goal-Driven Execution
- Define success criteria first.
- Write tests first. Then pass them.
- Declarative > imperative. Give goals, not steps.
